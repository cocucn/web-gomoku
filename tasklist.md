# AI 五子棋网页版任务清单

## 已规划任务

1. 创建 PRD 文档，明确功能边界和交付目标
2. 创建设计文档，定义页面结构、状态模型、AI 分级和禁手策略
3. 创建任务清单文档，沉淀实施步骤
4. 实现 `web-gomoku.html` 页面骨架与响应式结构
5. 实现 `styles.css`，完成桌面与移动端适配
6. 实现棋盘渲染与交互逻辑
7. 实现玩家先后手选择与新局初始化
8. 实现低级/中级/高级 AI 落子逻辑
9. 实现悔棋逻辑
10. 实现黑棋禁手检测开关与校验
11. 实现落子次序显示开关
12. 实现玩家托管能力与托管等级选择
13. 实现状态提示、最近一步高亮和结果展示
14. 本地验证页面可直接静态打开运行
15. 编写 `deploy.md`，说明 GitHub 与 GitHub Pages 发布流程

## 验收检查项

- 页面在桌面与手机视口下布局正常
- 可选择玩家先手或后手
- AI 三档难度可选并可正常落子
- 悔棋可恢复对局
- 黑棋禁手检测可开关
- 手数显示可开关
- 托管可开关且等级可选
- 主入口为 `web-gomoku.html`
- 文档完整可读
