# AI 五子棋网页版 PRD

## 1. 项目目标

实现一个可直接部署到 GitHub Pages 的单页网页版 AI 五子棋游戏，兼容 PC 和手机浏览器，支持基础对弈、AI 难度分级、禁手检测、悔棋、手数显示和玩家托管。

主入口文件固定为 `web-gomoku.html`。

## 2. 目标用户

- 想在浏览器里直接体验五子棋的普通用户
- 想快速部署静态网页到 GitHub Pages 的开发者
- 需要在手机和桌面设备上都可使用的用户

## 3. 核心功能

### 3.1 对弈基础

- 15x15 棋盘
- 玩家可选择先手或后手
- 玩家与 AI 对战
- 自动判定胜负
- 支持重新开始

### 3.2 AI 能力

- 低级 AI：偏随机、基础防守
- 中级 AI：启发式评分，兼顾进攻与防守
- 高级 AI：更高权重的攻防评估与更深层候选比较

### 3.3 规则控制

- 支持悔棋
- 支持开启/关闭黑棋禁手检测
- 开启禁手后，对黑棋执行禁手校验
- 禁手至少覆盖：长连、双三、双四

### 3.4 辅助显示

- 支持切换显示落子次序
- 次序数字显示在棋子中心
- 高亮最近一步
- 显示当前回合、对局状态、胜负信息

### 3.5 玩家托管

- 人类玩家一侧可开启 AI 托管
- 托管 AI 可单独选择低级/中级/高级
- 托管开启后，轮到玩家时由 AI 自动落子

## 4. 非功能要求

- 页面自动适配宽屏和窄屏设备
- 不依赖后端，可离线静态运行
- 操作反馈明确，按钮状态可理解
- 页面结构清晰，适合公开发布

## 5. 典型用户流程

1. 用户打开页面
2. 选择自己执黑或执白
3. 选择 AI 难度
4. 选择是否开启黑棋禁手
5. 点击开始对局
6. 在棋盘点击落子，或开启托管由 AI 自动落子
7. 过程中可悔棋、切换手数显示、重新开始
8. 对局结束后查看结果，继续新局

## 6. 成功标准

- 在桌面和手机上均可顺畅完成一局游戏
- AI 三档难度表现有明显差异
- 禁手开启时黑棋非法落子会被阻止并提示
- 悔棋逻辑稳定，不破坏对局状态
- GitHub Pages 可直接部署并访问
